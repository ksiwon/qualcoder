## 1. 프로젝트 소개

**QualCoder**는 사회과학·디자인·의료 분야의 질적 연구(Qualitative Research)를 위한 **오픈소스 브라우저 기반 분석 도구**입니다.

ATLAS.ti·NVivo 등 기존 상용 소프트웨어가 제공하는 핵심 기능을 웹 기술로 재구현하고, 여기에 **Gemini AI 기반 STT 전사**와 **AI 코드 추천** 기능을 더해 연구 워크플로 전체를 하나의 앱에서 처리할 수 있도록 설계되었습니다.

> **이 도구는 MEDial 프로젝트(KAIST)의 Formative Study를 위한 인터뷰 분석 과정에서 실제 사용 목적으로 개발되었습니다.**

### 핵심 철학

- **브라우저만 있으면 된다** — 설치 없이, 로컬에서, 데이터는 외부로 나가지 않는다.
- **전체 파이프라인** — 녹음 파일 → 전사 → Quotation → 코딩 → 분석 → 보고서 출력.
- **연구자 중심** — 모든 판단은 연구자가 내린다. AI는 보조할 뿐이다.
- **탭 간 연동** — 각 단계의 결과를 JSON/CSV로 내보내고 다음 단계에서 그대로 불러온다.

## 2. 주요 기능

| 카테고리 | 기능 |
|---|---|
| **전사 (STT)** | Gemini API 기반 오디오 → CSV 자동 전사, 화자 분리, 완성도 체크 |
| **문서 관리** | CSV / DOCX 업로드, 시간·화자·내용 3열 자동 파싱 |
| **Quotation** | 텍스트 드래그 → Quotation 생성, 마진 태그 표시, 하이라이트 |
| **코드 관리** | 코드 생성·편집·삭제, 카테고리·설명 관리, 색상 지정, 정렬 |
| **그룹화** | 코드 내 Quotation을 의미 단위로 그룹 분류, 드래그 이동 |
| **AI 보조** | 드래그 시 Gemini가 적합한 코드 자동 추천, 원클릭 적용 |
| **Board 뷰** | 코드별 카드 보드, 카테고리/키워드 필터 |
| **Network Graph** | 카테고리 → 코드 → 그룹 계층 관계도, 노드 자유 드래그 |
| **Query Tool** | AND / OR / NOT 불리언 조건으로 Quotation 검색 |
| **Co-occurrence** | 두 코드가 동시에 적용된 빈도 히트맵 |
| **Analytic Memos** | 코드·Quotation에 연구자 분석 노트 연결, 템플릿 제공 |
| **저장/불러오기** | localStorage 자동저장, JSON 내보내기/가져오기, XLSX 최종 출력 |
| **설정** | API 키, 기본 모델, 인터뷰어 레이블 등 영구 설정 관리 |

## 3. 시작하기

### 사전 요구사항

- **Node.js** 18 이상
- **npm** 또는 **yarn**
- (선택) Gemini API 키 — [Google AI Studio](https://aistudio.google.com) 에서 무료 발급

### 설치 및 실행

```bash
# 1. 압축 해제 후 디렉토리 이동
cd qualitative-tool

# 2. 의존성 설치
npm install

# 3. 개발 서버 실행
npm run dev
```

브라우저에서 `http://localhost:5173` 접속.

### 프로덕션 빌드

```bash
npm run build
# dist/ 폴더에 정적 파일 생성
# 어떤 정적 호스팅(Nginx, GitHub Pages, Vercel 등)에도 배포 가능
```

### 권장 첫 사용 순서

1. **⚙️ Settings** 탭 → Gemini API 키 입력 및 저장
2. **🎙️ STT** 탭 → 인터뷰 녹음 파일 업로드 및 전사
3. **📄 Docs** 탭 → 전사 CSV 업로드, 텍스트 드래그로 Quotation 생성
4. **🏷️ Codes** 탭 → 코드 정의·설명 작성, 그룹 구성
5. **📌 Board / 🕸️ Graph** 탭 → 패턴 시각적 탐색
6. **🔍 Query / ⬛ Matrix** 탭 → 심층 분석
7. **📝 Memos** 탭 → 분석 메모 기록
8. **TopBar → ↓ XLSX** → 최종 보고서 출력

### 🎙️ STT — 음성 전사

**목적:** 녹음 파일(.m4a / .mp4 / .mp3 / .wav)을 Gemini API로 자동 전사해 CSV를 생성합니다.

**사용 방법:**

1. Settings 탭에서 API 키를 먼저 등록하면 자동으로 연동됩니다.
2. 오디오 파일을 드래그 앤 드롭하거나 클릭해서 추가합니다.
3. 모델을 선택합니다 (기본: `gemini-3.1-flash-lite-preview`).
4. **▶ 전사 시작** 버튼을 클릭합니다.
5. 오른쪽 로그 창에서 실시간 진행 상황을 확인합니다.
6. 완료 후 결과 탭에서 시간·화자·내용을 미리보고 CSV 다운로드합니다.

**완성도 체크:**

파일 크기와 마지막 타임스탬프를 비교해 전사 완성도(%)를 자동 계산합니다. 70% 미만이면 경고 표시와 함께 재시도를 권장합니다.

**출력 CSV 형식:**

```
시간,화자,내용
00:03,인터뷰어,"안녕하세요. 오늘 인터뷰에 참여해 주셔서 감사합니다."
00:08,어르신,"어, 네. 반갑습니다."
```

> 다음 단계: 생성된 CSV를 📄 Docs 탭에 업로드합니다.

---

### 📄 Docs — 문서 관리 및 코딩

**목적:** 인터뷰 문서를 불러와 텍스트를 드래그해 Quotation을 만들고 코드를 연결합니다.

**지원 파일:**
- CSV (시간·화자·내용 컬럼) — STT 결과 또는 직접 작성한 전사본
- DOCX — mammoth.js로 텍스트 추출 후 타임스탬프/화자 자동 파싱
- JSON / .qualcoder — 프로젝트 파일 또는 코드북 파일

**Quotation 생성 방법:**

1. 문서 본문에서 코딩하고 싶은 텍스트를 마우스로 드래그합니다.
2. 팝업 메뉴가 나타납니다. 분석 메모를 입력하고 **✚ Quotation 생성**을 클릭합니다.
3. 이어서 코드 연결 팝업으로 자동 전환됩니다.

**코드 연결:**
- AI 추천 (API 키 설정 시): 발화 맥락을 분석해 적합한 코드를 자동 추천합니다.
- 수동 검색: 코드명을 검색하거나 목록에서 토글해 연결합니다.
- 새 코드 생성: 입력란에 코드명을 입력하고 Enter를 누르면 즉시 생성 후 연결됩니다.

**화면 구성:**

```
┌──────────────────────────────┬─────────────────────┐
│  시간  화자    발화 내용      │  Quotation 마진 태그 │
│  03:13 어르신  급할 때 못    │  [Capability]        │
│                받을, ...     │  병의원 수가 적어…   │
│  (드래그 → 하이라이트)       │                      │
└──────────────────────────────┴─────────────────────┘
```

오른쪽 마진 영역의 태그를 클릭하면 해당 Quotation의 코드 편집 팝업이 열립니다.

### 🏷️ Codes — 코드 관리자

**목적:** 코드 전체를 관리하고, 각 코드에 연결된 Quotation을 그룹별로 정리합니다.

**3-패널 레이아웃:**

```
┌──────────────┬──────────────────────────┬──────────────┐
│  코드 목록   │  Quotation Board         │  코드 상세   │
│  (좌)        │  (중앙)                  │  (우)        │
│              │                          │              │
│  [Capability]│  그룹: 시골 의료 자원    │  코드명 편집 │
│  [Caregiver] │  ┌────┐ ┌────┐ ┌────┐  │  설명 입력   │
│  [Necessity] │  │카드│ │카드│ │카드│  │  카테고리    │
│  ...         │  └────┘ └────┘ └────┘  │  색상        │
└──────────────┴──────────────────────────┴──────────────┘
```

**코드 생성:** + New Code 버튼 → 모달에서 키워드, 설명(Description), 분류(Category) 입력.

**정렬 옵션:**

| 모드 | 설명 |
|---|---|
| 분류별 | Category로 그룹핑해서 표시 |
| 이름순 | 가나다 순 |
| 빈도순 | Quotation이 많은 코드 우선 |

**그룹 관리:** 중앙 보드에서 + 그룹 추가 → Quotation 카드를 드래그해서 이동. 그룹 제목은 클릭 후 인라인 편집 가능.

**내보내기 / 가져오기:**
- ↓ 코드북 JSON: 코드 정의만 담은 파일 (다른 프로젝트에서 재사용 가능)
- ↓ 코딩 결과: 코드 + Quotation 전체 데이터
- ↑ 코드북 가져오기: JSON에서 코드 일괄 불러오기 (중복 건너뜀)

---

### 📌 Board — 코드 보드

**목적:** 전체 코드를 칸반 보드 스타일로 한눈에 조망합니다.

- 각 코드가 하나의 **열(Column)**로 표시됩니다.
- 열 헤더에 코드 설명, 본문에 연결된 Quotation 카드가 나열됩니다.
- 카테고리 필터 칩으로 원하는 분류만 표시합니다.
- 키워드 검색으로 코드명·설명을 필터링합니다.
- 카드를 클릭하면 해당 문서의 원문으로 이동합니다.

---

### 🕸️ Graph — 네트워크 그래프

**목적:** 코드 간 계층 관계(카테고리 → 코드 → 그룹)를 시각적 그래프로 탐색합니다.

| 노드 | 의미 |
|---|---|
| 둥근 채색 노드 | 카테고리 (Category) |
| 직사각형 (연한 색) | 코드 (Code) |
| 소형 직사각형 | 그룹 (Group) |

**조작 방법:**
- 노드 드래그: 자유롭게 위치 이동
- 배경 드래그: 캔버스 패닝
- 마우스 휠: 줌 인/아웃
- 🔄 자동 배치: 계층 구조 기준으로 재배치
- 연결선 토글: 엣지 표시/숨김

이론 포화(Theoretical Saturation) 단계에서 코드 간 관계를 시각적으로 검토하고 선택적 코딩(Selective Coding) 작업에 활용합니다.

---

### 🔍 Query — 불리언 쿼리

**목적:** 복수 코드의 AND / OR / NOT 조합으로 특정 조건을 만족하는 Quotation을 검색합니다.

**쿼리 예시:**

```
START  "[Capability] 진료 대기 시간"
AND    "[Caregiver] 간병인 부재"
NOT    "[Necessity] 기술 필요성"
```

위 조건은 "진료 대기 시간 코드가 붙었고, 간병인 부재 코드도 붙었지만, 기술 필요성 코드는 없는 Quotation"을 찾습니다.

결과를 문서별·화자별로 필터링할 수 있습니다.

---

### ⬛ Matrix — Co-occurrence Matrix

**목적:** 두 코드가 동시에 하나의 Quotation에 적용된 횟수를 히트맵으로 시각화합니다.

- 색상이 진할수록 두 코드의 공동 출현 빈도가 높습니다.
- 최소 co-occurrence 임계값을 설정해 낮은 빈도를 걸러냅니다.
- 셀을 클릭하면 해당 두 코드가 동시에 적용된 Quotation 목록이 하단에 표시됩니다.
- Quotation 클릭 시 원문 문서로 이동합니다.

높은 co-occurrence는 두 개념이 함께 나타나는 맥락이 있음을 시사합니다. 예를 들어 `[Capability] 진료 대기 시간`과 `[Caregiver] 간병인 부재`의 co-occurrence가 높다면, "혼자 아픈 상황에서 오래 기다려야 하는 이중 부담"이라는 패턴을 발견할 수 있습니다.

---

### 📝 Memos — Analytic Memos

**목적:** 코딩 과정에서 연구자의 해석, 가설, 패턴 발견을 기록하는 분석 메모 공간입니다.

**주요 기능:**
- 자유 형식 텍스트 편집
- 특정 코드 및 Quotation에 메모 연결
- 템플릿 삽입: 초기 인상 / 패턴 발견 / 가설 / 반증 사례 / 이론 연결

**템플릿 예시:**

```markdown
## 패턴 발견

이 코드에서 반복적으로 나타나는 패턴:
- 시골 지역 어르신들은 병원까지의 거리보다 대기 시간을 더 큰 문제로 인식함
- 특히 혼자 거주하는 경우 이동 자체의 어려움이 복합적으로 작용함

## 가설

만약 AI 원격 진료 시스템이 도입된다면, 이동 부담보다
'내 이야기를 들어줄 의사'에 대한 신뢰 문제가 더 큰 장벽이 될 것이다.

## 반증 사례

박창호 어르신: 오히려 직접 대면을 선호함.
기술에 대한 거부감이 아니라 관계적 돌봄에 대한 욕구가 더 강함.
```

### ⚙️ Settings — 설정

**목적:** API 키, 기본 모델, 코딩 환경을 관리합니다. 설정은 브라우저 localStorage에 영구 저장됩니다.

| 설정 항목 | 설명 | 기본값 |
|---|---|---|
| Gemini API Key | STT·AI 보조 기능에 사용 | (없음) |
| 기본 모델 | STT 및 AI 추천에 사용할 모델 | `gemini-3.1-flash-lite-preview` |
| 기본 카테고리 | 새 코드 생성 시 기본 분류 | `사용자 경험` |

```
🎙️ STT  ──→  📄 Docs  ──→  🏷️ Codes  ──→  📌 Board
  │             │              │
 .csv 출력    .csv 입력      .json 출력
              .json 입력     .json 입력

🕸️ Graph  ←──  🔍 Query  ←──  ⬛ Matrix
                                       │
                                  📝 Memos
                                       │
                                  ↓ XLSX 최종 출력
```

**파일 형식별 역할:**

| 파일 | 생성 위치 | 사용 위치 | 내용 |
|---|---|---|---|
| `*_전사결과.csv` | STT 탭 | Docs 탭 | 시간·화자·내용 전사 데이터 |

| `qualcoder_export_*.xlsx` | TopBar | 외부 분석 도구 | 코드북·Quotation·매핑 시트 |

## 6. AI 보조 기능

### 동작 방식

Gemini API 키가 설정된 경우, 텍스트를 드래그해 Quotation을 생성하면 자동으로 AI 코드 추천이 실행됩니다.

```
연구자가 텍스트 드래그
        ↓
QualCoder가 아래 정보를 Gemini에 전달:
  • 선택된 발화 텍스트
  • 현재 프로젝트의 기존 코드 목록 (최대 30개)
        ↓
Gemini가 적합한 코드 최대 4개 추천
  (기존 코드 재사용 + 새 코드 제안)
        ↓
팝업에 추천 칩으로 표시 → 클릭으로 즉시 적용
```

### AI가 보조하는 시점

AI 보조는 유의미한 성과를 낼 수 있는 특정 단계에만 적용됩니다.

| 기능 | 적용 이유 |
|---|---|
| **코드 추천** | 수백 개의 Quotation 처리 시 일관성 있는 코드 선택 지원 |
| **STT 전사** | 수동 전사 대비 시간 90% 절감, 화자 분리 자동화 |

그룹화, 메모 작성, 이론 구성 등 해석적 판단이 필요한 작업은 AI가 개입하지 않습니다. 질적 연구의 핵심은 연구자의 성찰적 사고이기 때문입니다.

변경사항이 생기면 상단바에 **"● 미저장"** 표시가 나타납니다. `Ctrl+S` 또는 사이드바 💾 버튼으로 저장하면 **"✓ 저장됨"** 으로 바뀝니다. 다음에 앱을 열면 마지막 저장 상태가 자동으로 복원됩니다.

| ↓ 코드북 JSON | `codebook_*.json` | 코드 정의 재사용 |
| ↓ 코딩 결과 | `coding_result_*.json` | 부분 공유, 단계적 백업 |

| 시트 | 내용 |
|---|---|

## 11. 설계 배경 및 방법론

### 질적 연구 코딩 프로세스

이 도구는 Grounded Theory와 Thematic Analysis의 표준 절차를 따릅니다.

**1단계 — 개방 코딩 (Open Coding)**
- 텍스트를 드래그해 Quotation 생성, 코드 연결
- AI 추천으로 초기 코드 집합 구성
- 도구: 📄 Docs 탭

**2단계 — 축 코딩 (Axial Coding)**
- 코드 그룹화로 하위 범주 구성
- Co-occurrence Matrix로 코드 간 관계 탐색
- Query Tool로 교차 패턴 검증
- 도구: 🏷️ Codes 탭, ⬛ Matrix 탭, 🔍 Query 탭

**3단계 — 선택적 코딩 (Selective Coding)**
- Network Graph로 핵심 범주 중심의 이론 구성
- Analytic Memos에 이론 발전 과정 기록
- 도구: 🕸️ Graph 탭, 📝 Memos 탭

**4단계 — 이론 포화 (Theoretical Saturation)**
- 새로운 코드가 더 이상 나타나지 않을 때까지 반복
- XLSX 내보내기로 최종 결과 정리

## 12. 자주 묻는 질문

**Q. Gemini API 키는 어떻게 발급하나요?**

[Google AI Studio](https://aistudio.google.com) 에 접속해 Google 계정으로 로그인 후 **Get API Key** 버튼을 클릭합니다. 무료 티어로도 충분히 사용 가능합니다.

**Q. 데이터가 외부 서버에 저장되나요?**

아니요. 프로젝트 데이터는 브라우저의 localStorage에만 저장됩니다. AI 기능 사용 시 해당 텍스트만 Google Gemini API로 전송되며, API는 요청 데이터를 학습에 사용하지 않습니다 (Google API 정책 기준).

**Q. 여러 사람이 동시에 작업할 수 있나요?**

현재 버전은 단일 사용자 도구입니다. 협업은 프로젝트 JSON 파일을 내보내고 팀원에게 공유한 뒤, 팀원이 가져오기하는 방식으로 진행합니다.

**Q. 음성 파일이 너무 크면 전사가 안 되나요?**

Gemini API의 파일 업로드 제한을 초과하면 오류가 발생합니다. 긴 인터뷰는 편집 소프트웨어로 분할 후 각각 전사하세요.

**Q. CSV 파일의 컬럼명이 다르면 어떻게 되나요?**

`시간`, `화자`, `내용` 컬럼명을 자동으로 인식합니다. 컬럼명이 다를 경우 빈 값으로 처리됩니다. CSV를 직접 편집해 헤더를 맞춰주세요.

**Q. DOCX 파일에 타임스탬프가 없으면 어떻게 처리되나요?**

타임스탬프 없이 화자:내용 형식만 있어도 파싱됩니다. 시간 컬럼이 빈 값으로 처리되고 나머지 내용은 정상 표시됩니다.

**Q. 전사 완성도 70%는 어떻게 계산하나요?**

파일 크기(MB)를 기준으로 예상 재생 시간을 추산하고, 마지막 타임스탬프와 비교합니다. 정확한 오디오 길이 분석이 아니므로 참고 지표로 활용하세요.

**Q. 키보드 단축키가 있나요?**

| 단축키 | 기능 |
|---|---|
| `Ctrl + S` | 프로젝트 저장 |
| 텍스트 드래그 | Quotation 생성 팝업 열기 |
| `Enter` | 코드 추가 확인 |
| `Esc` | 팝업 닫기 |
| 마우스 휠 | 네트워크 뷰 줌 |
| 배경 드래그 | 네트워크 뷰 패닝 |
